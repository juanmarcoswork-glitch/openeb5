P1 · Cache-Control no-store + cookie PHPSESSID por plugin Booked
CriticalClaudeNo design
15 min

Problema. Plugin Booked v3.0.0 ejecuta session_start() global en booked.php:368 → cookie PHPSESSID + Cache-Control: no-store → ninguna capa de caché (NitroPack / AccelerateWP / NGINX) funciona.

Solución. Snippet add_filter('booked_sessions_enabled', '__return_false') en child theme. Más importante: NitroPack drop-in advanced-cache.php sirve HIT antes de PHP — ese es el verdadero desbloqueo.

Verificado 11 Jun. curl -I https://openeb5.com/ → Cache-Control: no-cache (ya no no-store) · X-Nitro-Cache: HIT · X-Nitro-Cache-From: drop-in · sin Set-Cookie: PHPSESSID. Nota técnica: el filtro Booked es no-op en la primera llamada (lógica del if) pero inofensivo. El drop-in NitroPack hace el trabajo real. Ver reports/A1-verificacion-cache.md.

A2 · Regresión payload mobile: 5.9 MB → 12 MB + 76 scripts cargados
CriticalConfigClienteNo design
10 min panel

Problema. El HTML cacheado lleva 76 scripts individuales (con prefijo render-blocking-nitro-min-*): NitroPack los identifica como render-blocking pero NO los combina ni difiere. Render-blocking mobile = 22.9 s.

Causa raíz. Tres hipótesis: (a) Combine JS: OFF en NitroPack, (b) solapamiento con AccelerateWP sin apagar, (c) WebP background no aplicado a CSS backgrounds.

Status 11 Jun. Diagnóstico completo en reports/A2-payload.md. Acción pendiente cliente (panel, no FTP): NitroPack → JavaScript → Combine JS ON · Defer JS Standard · Delay JS OFF + Panel hosting → AccelerateWP/LSCache/NGINX FastCGI OFF. Esperado: 76 → 25–35 scripts, payload 12 → 5-7 MB.
